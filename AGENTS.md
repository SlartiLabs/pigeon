# AGENTS.md ‚Äî canonical context for `pigeon` (formerly `agentctx`)

> **This is the single source of truth.** `CLAUDE.md` and `GEMINI.md` are
> generated pointers to this file and must never be edited by hand. Run
> `pigeon refresh` (or `make context`) after editing this file.

## Goal

`pigeon` (the `agentctx` command remains as an alias) lets multiple AI runtimes ‚Äî Claude Code, Gemini CLI, Codex ‚Äî and
their sub-agents **share project context and hand work to each other
efficiently**: minimal tokens, no re-transmission of state, validated messages.
It is a small glue layer designed to live inside any repository.

## Architecture ‚Äî three decoupled layers

1. **Canonical context** (this file). Plain repo files every CLI already reads,
   single-sourced so there is no drift. `CLAUDE.md` / `GEMINI.md` are generated
   thin pointers here ‚Äî they contain no duplicated prose, so they cannot drift.
2. **Handoff contract** (`.pigeon/handoff.schema.json` + `pigeon handoff`).
   A JSON-Schema-validated message carrying **sparse state deltas plus pointers
   ‚Äî never payloads**. Separate CLIs share no memory; the contract is the
   filesystem, not anyone's context window. Handoffs are validated on receipt
   and appended to `.pigeon/handoffs/`.
3. **Retrieval** (`pigeon retrieve`). Hybrid lexical (ripgrep) + BM25 over the
   repo and the generated manifest. Vector and graph are deferred.

The guiding rule: **start simple, measure token cost, and only graduate to
heavier machinery where measurement proves it is the bottleneck.**

## Decision record (why these choices)

- **Graph (Graphiti / GraphRAG) is deferred.** These repos are fast-churn
  (idea ‚Üí pilot in weeks, state changing every few minutes). A knowledge graph
  pays off only when the read-to-write ratio is high; its LLM-based ingestion
  cost is wasted on state that is constantly rewritten. Revisit only for a
  stable core, gated on metrics.
- **JSON, not YAML, for the contract.** Every model's tool-calling is trained
  heavily on JSON; YAML's significant whitespace is a cross-model failure mode.
  (Config is YAML because it is human-edited; that is the one exception.)
- **Pointers need a resolver.** Any agent receiving `repo://‚Ä¶`, `file://‚Ä¶`, or
  `s3://‚Ä¶` must be able to resolve it; `agentctx` ships a common resolver, so
  pointers are never an assumption.
- **Tooling language: Python 3.11+.** Minimal, pinned deps.

increasing ambition (as of 2026-06-14):

- **Rung 1 ‚Äî the edit‚Üíreview‚Üíverify‚Üíconcord spine: validated.** Worktree
  isolation, full-diff materialization to the shared tree, `receives:` pointer

(`.pigeon/memory/decisions.md`).

## Repo map

The authoritative, machine-readable map is the generated manifest at
`.pigeon/manifest.json` (modules, public interfaces, entry points, decisions,
owners). It is deterministic and regenerated by `pigeon refresh`. **Read the
manifest instead of dumping the file tree.**

Source layout:

- `src/pigeon/manifest.py` ‚Äî build the deterministic project manifest.
- `src/pigeon/context.py` ‚Äî generate `CLAUDE.md` / `GEMINI.md` from this file.
- `src/pigeon/handoff.py` ‚Äî build, validate, serialize, append handoffs.
- `src/pigeon/resolve.py` ‚Äî resolve pointers (`repo://`, `file://`, paths; `s3://` flagged).
- `src/pigeon/retrieval.py` ‚Äî hybrid ripgrep + BM25 retrieval.
- `src/pigeon/tokens.py` ‚Äî token counting + `.pigeon/metrics.jsonl` accounting.
- `src/pigeon/cli.py` ‚Äî the `pigeon` command (`agentctx` kept as an alias).
- `src/pigeon/coordinate.py` ‚Äî parallel fan-out: DAG scheduler, worktrees, budgets, run manifests.
- `src/pigeon/distill.py` ‚Äî episodic log -> committed session memory + decision ledger.
- `src/pigeon/graph.py` ‚Äî derived entity graph (`[[wiki-links]]` + handoff provenance).
- `src/pigeon/pack.py` ‚Äî bounded pre-task context bundles.
- `src/pigeon/skills.py` ‚Äî project playbooks into runtime-native subagent files.
- `src/pigeon/mcp_server.py` ‚Äî the contract as MCP tools (stdio).

The on-disk contract directory is `.pigeon/` in pigeon-native repos.
The legacy `.pigeon/` name is honored forever: discovery prefers
`.pigeon/`, falls back to `.pigeon/`, and `pigeon init` never migrates an
existing legacy dir ‚Äî the dirname is the contract, deployed consumers never
break.

## How agents should behave

- **Retrieve, don't dump.** Use `pigeon retrieve "<query>"` to pull bounded,
  ranked slices. Do not paste whole files or the repo tree into your context.
- **Hand off via the contract.** To pass work to another runtime or sub-agent,
  emit a handoff validated against the schema; carry pointers, not payloads.
- **Resolve pointers on demand.** Use the shared resolver; do not inline
  artifact contents into a handoff.
- **Measure.** Every handoff and retrieval is token-accounted. Prefer the
  approach `pigeon metrics` shows to be cheaper; do not assume savings.

## Conventions

- Python 3.11+, fully type-hinted; `pytest` for tests. Pinned deps, minimal set.
- Deterministic outputs where claimed (manifest is byte-stable for fixed input).
- Append-only logs: handoffs and metrics are never rewritten in place.
